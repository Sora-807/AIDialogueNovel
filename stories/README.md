# 故事源数据格式说明

本目录存放故事的不可变源数据。所有 `.md` 文件使用 YAML frontmatter（`---` 包裹的 YAML 头）。

**所有 `description` 字段遵循同一语义**：渐进式披露描述。任何 agent 在做概览/列表时都可以看到并使用它——它是这条数据自带的摘要，谁需要就能读。

---

## 目录结构

```
stories/{故事名}/
  story.json              ← 故事元信息
  世界观/                  ← 世界观条目
    {条目名}.md
    {子目录}/{条目名}.md
  大纲/                    ← 大纲文件
    【1】{章节名}.md
    【2】{章节名}.md
  characters/{角色名}/
    profile.md             ← 全知设定（Author 参考资料）
    initial_state.md       ← 初始状态（注入角色 agent system prompt）
```

---

## story.json

```json
{
  "story_id": "demo",
  "name": "显示名称",
  "description": "渐进式披露描述",
  "author": "作者名"
}
```

| 字段 | 必填 | 说明 |
|------|------|------|
| story_id | 是 | 内部标识，与目录名一致 |
| name | 是 | 显示名称 |
| description | 否 | 渐进式披露描述 |
| author | 否 | 作者 |

---

## 世界观条目（.md）

**路径**：`世界观/` 下任意 `.md` 文件，支持子目录。相对路径（去掉 `.md` 后缀）为条目标识。

**YAML frontmatter**：

```yaml
---
tags: [public, 魂师体系]    # 必填。public = Narrator 自动可见，hidden = 需 Author 授权
name: 显示名称               # 必填
description: 渐进式披露描述   # 必填，一句话简述
---
```

正文自由组织。

**权限**：Author 看全部；Narrator 看 `public` + 已授权的 `hidden`；角色看已告知条目。

---

## 大纲文件（.md）

**路径**：`大纲/` 下按 `【数字】章节名.md` 命名，数字从 1 开始连续。

**YAML frontmatter**：

```yaml
---
name: 开学日          # 必填：章节显示名
description: 概述      # 必填：渐进式披露描述
---
```

正文自由组织（如 `## 场景目标`、`## 关键节拍`）。Author 的大纲概览中展示所有章节名 + 当前章节的 `description`。

---

## 角色文件（两个文件）

每个角色目录下有两个 `.md` 文件，职责分离：

| 文件 | 职责 | 读者 | 是否可变 |
|------|------|------|----------|
| `profile.md` | 全知设定——角色的完整背景、隐藏秘密、命运走向 | 仅 Author | 不变 |
| `initial_state.md` | 初始状态——角色自己知道的信息 | 注入角色 agent system prompt | 不变（运行时被 save 覆盖） |

**运行时状态优先级**：`saves/{故事}/characters/{角色}/state.md` > `initial_state.md`

---

### profile.md — 全知设定

```yaml
---
name: 显示名称
description: 全知版渐进式披露描述（包含身份、因果、主要关系、隐藏命运的浓缩。不担心剧透——Author 需要这些信息来规划剧情）
---
```

正文为 `# 全貌设定`，按需组织章节（如 `## 身世背景`、`## 武魂与能力`、`## 性格深度分析`、`## 关键人际关系`、`## 神位之路`）。**不写发言风格和行为习惯**——这些属于 `initial_state.md`。

**原则**：profile 是 Author 的"编剧参考资料"。任何幕后信息都往这里放——因果、命运、隐藏身份、人物最终结局。不怕剧透，怕 Author 不知道。

---

### initial_state.md — 初始状态

注入角色 agent system prompt。用固定 section 组织，每个 section 以 `<!-- 指引 -->` 开头告诉 agent 如何填写（后续角色自己更新状态时也需要遵循这个格式）。

```markdown
# 人物状态

## 基本信息
<!-- 姓名、性别——通常不变，仅做记录 -->

## 当前修为
<!-- 格式：称号（等级，几环）。武魂和魂环信息跟在后面 -->

## 身世背景
<!-- 角色自己知道的过往，1-3句关键信息。不写角色不知道的事 -->

## 外貌
<!-- 当前的外貌描述，有变化时更新 -->

## 心理状态
<!-- 当前的内心状态，1-3句。包含情绪、目标、担忧 -->

## 发言风格
<!-- 说话的语气、习惯、句式示例。有变化时更新 -->

## 行为习惯
<!-- 独处时的习惯动作、下意识的反应等。有变化时更新 -->

## 对他人的看法
<!-- 格式：- 角色名：看法。有变化时增删改 -->

## 公开信息
<!-- 外界可以看到的关于你的信息。只写可见的外在事实，不写内心秘密 -->
```

**填写原则**：
1. 只写角色自己知道的事——身世背景不能超出角色的真实认知边界
2. `<!-- 指引 -->` 是在告诉 agent "这个 section 怎么填、格式是什么"——后续角色更新 state 时会参考这些指引
3. 发言风格必须带句式示例——否则 agent 不知道"怎么说话"
4. 对他人的看法只填已见面/已知晓的角色，不要预填未出场角色
5. 公开信息是外界视角——Narrator 和其他角色看到的你
